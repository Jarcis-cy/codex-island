# Codex Island v0.0.2

[English](#english) | [中文](#中文)

## English

Compared with `v0.0.1`, `v0.0.2` focuses on making remote Codex sessions easier to inspect and safer to switch, while also fixing several session loading issues that could leave the app in a confusing state.

### New Features

- Remote chat headers now show the active model, reasoning effort, service tier, and remaining context percentage.
- Local chat headers now show the active model and remaining context percentage parsed from transcript events.
- Remote threads now support a more reliable `/new` flow so you can explicitly start a fresh thread even when another thread exists under the same SSH target and working directory.
- Remote `/resume` now reads from the raw thread directory, which makes it much easier to reopen older same-directory threads.
- Remote diagnostics can now stay off by default and only write `remote-app-server.jsonl` after you enable `Remote Debug Logs` from the menu.

### Bug Fixes

- Fixed remote session rebinding after `/new`, so switching to a fresh thread no longer snaps back to a newer same-directory thread.
- Fixed remote open-session hydration and error handling so existing threads load more consistently.
- Fixed the first local `Open Session` flow that could open an empty placeholder chat before transcript history was ready.
- Fixed delayed local transcript loading and local chat rebinding, reducing blank or stale chat state on first load.
- Fixed local interaction input handling for multi-question prompts and improved remote slash-command compatibility with recent Codex app-server responses.

### Notes

- Release build version: `0.0.2 (2)`
- This release continues to support both local hooks-based sessions and SSH remote hosts running `codex app-server`.

### Install

1. Download `CodexIsland-0.0.2-macOS-unsigned.zip`.
2. Unzip it and move `Codex Island.app` to `Applications`.
3. Launch the app manually.

### Release Notes

- This build is unsigned and not notarized because this environment does not have Developer ID signing, notarization credentials, or Sparkle release keys configured.
- macOS may show security warnings on first launch. Use the standard Gatekeeper override flow if you trust the build.

## 中文

相对于 `v0.0.1`，`v0.0.2` 主要补强了远端 Codex 会话的可见性与切换稳定性，同时修掉了几处会让会话打开后表现异常的加载问题。

### 新特性

- 远端聊天头部现在会显示当前模型、reasoning effort、service tier，以及剩余上下文百分比。
- 本地聊天头部现在会根据 transcript 事件显示当前模型和剩余上下文百分比。
- 远端线程新增更可靠的 `/new` 流程，即使同一个 SSH target 和工作目录下已有线程，也可以显式新开一个 thread。
- 远端 `/resume` 现在基于 raw thread 目录取候选，重新打开同目录旧线程更稳定。
- 远端诊断日志现在默认关闭，只有在菜单里手动开启 `Remote Debug Logs` 后才会写入 `remote-app-server.jsonl`。

### Bug 修复

- 修复远端 `/new` 之后的会话 rebinding 问题，切到新 thread 后不再被同目录较新的 thread 自动顶回。
- 修复远端已有会话的打开、hydration 和错误处理，现在线程恢复更稳定。
- 修复本地首次 `Open Session` 可能先出现空白占位聊天窗口的问题。
- 修复本地 transcript 延迟到达时的聊天 rebinding 与历史加载问题，减少首次进入时的空白或过期状态。
- 修复本地多问题交互输入处理，以及远端 slash command 对较新 Codex app-server 响应格式的兼容性问题。

### 说明

- 本次发布版本号：`0.0.2 (2)`
- 本版本继续同时支持本地 hooks 会话和通过 SSH 连接的远端 `codex app-server`。

### 安装

1. 下载 `CodexIsland-0.0.2-macOS-unsigned.zip`。
2. 解压后把 `Codex Island.app` 移到 `Applications`。
3. 手动启动应用。

### 发布说明

- 当前构建仍然是未签名、未公证版本，因为这台环境没有配置 Developer ID 签名、公证凭据或 Sparkle 发布密钥。
- macOS 首次启动时可能会给出安全提示；如果你信任这份构建，可以按标准 Gatekeeper 放行流程处理。
